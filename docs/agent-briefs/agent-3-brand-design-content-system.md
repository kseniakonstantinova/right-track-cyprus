## Core Brand Direction You Must Respect
The guide must clearly reflect the brand book, including:

### Positioning
- Right Track is `The Athlete's Physio`
- performance-focused rehabilitation
- bridges the gap between traditional physiotherapy and athletic training
- not wellness/spa
- not generic hospital tone
- not a soft lifestyle brand

### Audience
- competitive athletes
- active individuals
- sports teams
- post-surgical patients with return-to-sport goals

### Brand Values / Personality
- strength
- recovery
- performance
- movement
- professionalism
- trust
- discipline
- return to sport

### Brand Voice
- expert
- confident
- motivational
- coach + expert

The brand book explicitly suggests:
- knowledgeable but accessible
- direct and assured
- inspiring action and progress
- "we speak like a coach who happens to be a medical expert"

Your guide must operationalize that.

### Visual System
The guide must reflect the brand book's visual direction:
- primary navy `#0A1628`
- primary orange `#FF6B35`
- supporting midnight blue `#1E3A5F`
- off-white `#F8F9FA`
- light gray `#E5E7EB`
- success green `#22C55E`
- warning amber `#F59E0B`

And also:
- navy as authority base
- orange for CTA and emphasis
- high contrast
- strong typographic hierarchy
- dramatic athletic imagery
- avoid spa / pastel / overly clinical hospital aesthetic

## Core Goal
Create a practical operating guide that ensures future site pages and decks feel like one system.

The guide should prevent:
- mixed visual language
- mixed brand tone
- CTA inconsistency
- confused page hierarchy
- partner-facing ambiguity
- resource pages that look detached from the brand

## What the Guide Must Cover

### 1. Purpose
Explain:
- who the guide is for
- what it governs
- that it applies to both web pages and presentation materials
- how it complements `docs/seo-guidelines.md`

### 2. Brand Translation
Turn the brand book into practical rules for digital implementation.

This section should answer:
- what "The Athlete's Physio" means in web copy
- what "performance-focused rehabilitation" means in page framing
- how to sound expert without sounding cold
- how to sound motivational without sounding generic

### 3. Naming and Messaging Rules
Define how the brand should be represented consistently across:
- page titles
- visible headings
- metadata
- CTA copy
- profile/about text
- partner pages

You do not need to rewrite site metadata, but you must define the rules.

### 4. Page Families
Document the main page families:
- homepage
- service page
- blog article
- clinician/profile page
- community / partnership page
- seminar / resource page
- campaign / promo page

For each:
- primary user intent
- primary CTA type
- what belongs above the fold
- what must be secondary
- tone considerations

### 5. Hero System
Create hero rules for each major page type.

The guide must explain:
- what a strong Right Track hero looks like
- how to balance authority and athletic energy
- what should never dominate the hero
- how to handle completed seminars and downloadable resources
- how to handle partner / community pages without confusing the visitor

### 6. CTA Hierarchy
Define:
- primary CTA
- secondary CTA
- tertiary / text CTA

Include rules such as:
- one clear primary CTA per page
- orange is used intentionally for primary emphasis
- duplicate CTAs to the same destination should be avoided unless context differs
- guide pages should privilege download / view materials actions
- partner pages should privilege contact / discuss collaboration actions

### 7. Card and Section System
Document reusable logic for:
- homepage promos
- community cards
- blog promos
- case / partnership cards
- speaker blocks
- offer blocks
- download blocks

This is especially important because current site work is page-by-page and tends to drift.

### 8. Badge Taxonomy
Create a badge system aligned with both brand and content clarity.

Examples to cover:
- Free Resource
- Seminar Guide
- Completed Seminar
- Partnership
- Community Event
- Research
- New

Explain when a badge helps clarity and when it creates confusion.

### 9. Typography and Hierarchy
Use the brand book's typography intent.

The guide must address:
- heading character
- body readability
- hierarchy from H1 to caption
- practical multilingual fallback strategy

Important:
- the brand book proposes Latin-focused type pairings and a multilingual fallback using Noto Sans
- the site has English and Greek pages
- the guide must explain how to preserve hierarchy and brand feel across EN and EL, even when the exact brand-book Latin pairings are not available everywhere

### 10. Color and Contrast Rules
Translate the brand palette into practical usage rules:
- background usage
- CTA usage
- section contrast
- supporting accents
- success / warning semantics

Include "do not" guidance:
- no soft pastel wellness palette
- no washed-out low-contrast sections
- no random color expansion

### 11. Photography and Imagery
The brand book is clear here.

Document:
- desired image mood
- what to capture
- what to avoid
- how photos should support sports/rehab credibility

Make this directly usable for:
- page banners
- team images
- promo assets
- deck slides

### 12. Graphic Elements
Turn the brand book's graphic cues into practical system guidance:
- dynamic lines
- directional arrows
- highlight boxes
- numbered markers
- body-part indicators
- icon usage

The point is not to reproduce the brand book slide by slide.
The point is to define how these elements should appear consistently across future work.

### 13. Presentation / Deck Rules
This section is mandatory.

The user explicitly noted that presentations are starting to need a guide too.

Document a lightweight system for:
- cover slide
- section divider slides
- educational content slides
- exercise slides
- data/stat slides
- closing/contact slides

Also define:
- tone of slide headlines
- image treatment
- icon treatment
- layout density
- how website and deck systems should feel related

### 14. SEO Alignment
Add a short section linking design/content choices to SEO rules.

This must include:
- one page, one clear intent
- no mixed-intent hero
- metadata should reflect visible page purpose
- page structure should support search intent
- internal CTA language should reinforce the canonical page role

Do not duplicate the full SEO guide.
Reference it and explain how the guides work together.

### 15. Do / Don't
End with sharp, implementation-level guidance based on current site realities.

Examples:
- do make seminar pages feel like complete resources after the event
- don't mix event-registration and guide-download intent
- do make community pages partner-readable
- don't let resource pages feel detached from collaboration context
- do use orange as intentional emphasis
- don't turn the brand into spa aesthetics

## Style Requirements
The document should be:
- practical
- high-signal
- easy for agents to apply
- grounded in the actual brand book and current site

Prefer:
- short sections
- direct rules
- examples
- implementation language

Avoid:
- generic branding fluff
- vague motivational copy
- abstract design philosophy detached from real page-building work

## Acceptance Criteria
Your work is done when:
- `docs/site-design-and-content-guide.md` exists
- it clearly reflects the brand book
- it clearly complements `docs/seo-guidelines.md`
- it is useful for both website pages and presentation decks
- it gives concrete rules for community pages, seminar/resource pages, CTA hierarchy, visual tone, and multilingual execution

When done, report:
- the file you created

- the most important guidance for community and seminar/resource pages
