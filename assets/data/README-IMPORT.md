# Импорт исторических данных в Firebase

Этот файл содержит инструкции по одноразовому импорту исторических данных за январь 2026 в Firebase.

## Данные для импорта

- **Файл:** `january-2026-complete-extended.json`
- **Период:** 6 января - 1 февраля 2026 (Weeks 2, 3, 4, 5)
- **Количество:** 133 бронирований
- **Терапевты:** Antonis (91 сессий), Charalambos (42 сессии)

## Инструкция по импорту

### Шаг 1: Откройте админ-панель
Откройте в браузере: `/pages/admin/index.html`

### Шаг 2: Перейдите в раздел Statistics
Нажмите на "Statistics" в боковом меню

### Шаг 3: Нажмите кнопку импорта
Нажмите на синюю кнопку **"Import January Data"**

### Шаг 4: Дождитесь завершения
Кнопка покажет прогресс:
```
Importing... 10
Importing... 20
...
```

После завершения появится сообщение:
```
Import complete!
133 bookings imported successfully.
```

Кнопка автоматически исчезнет, чтобы предотвратить повторный импорт.

### Шаг 5: Просмотрите данные
Данные автоматически появятся в системе - обновлять страницу не нужно!

## Проверка импорта

После импорта в разделе **Statistics** выберите:
- Period: **January 2026**
- View Type: **Monthly Summary**

Вы должны увидеть:
- **Antonis:** 91 sessions | €2,146
- **Charalambos:** 42 sessions | €1,007.5
- **Total:** 133 sessions | €3,153.5

## Важно

- Этот импорт нужно выполнить **только один раз**
- Данные сохраняются в Firebase и будут доступны всегда
- Если импорт был выполнен дважды, будут дубликаты (используйте фильтры в Bookings для удаления)
- Все импортированные бронирования имеют статус `confirmed`
- Источник всех бронирований отмечен как `historical-import`

## Структура данных

Каждое бронирование содержит:
- Client name, phone, email
- Service, Therapist
- Date, Time slot
- Payment type (gesy-new, gesy-old, private)
- Location (gym, physio, home)
- Price (EUR)
- Notes
- Status (confirmed)

## После импорта

После успешного импорта вы можете:
1. **Просматривать** все бронирования в разделе Bookings
2. **Редактировать** любое бронирование кнопкой "Edit"
3. **Фильтровать** по терапевту, дате, типу платежа
4. **Анализировать** в разделе Statistics
